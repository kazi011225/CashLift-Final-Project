# CashLift

CashLift is a donation tracking app that uses AI to explain, in plain language, what a donation actually does in the real world. Instead of giving to a cause and never knowing what happened next, users log a donation and get a short, concrete explanation of its impact, generated by Google's Gemini API.

Built as a final project to practice a full stack flow: authentication, a relational database design, server side API integration, and deployment.

## Features

- Account creation with hashed passwords (bcryptjs), no plain text passwords are ever stored
- Session based login and logout, with protected routes that redirect to sign in if not authenticated
- A dashboard where a signed in user can:
  - Log a donation by choosing a cause and entering an amount
  - Automatically receive an AI generated explanation of that donation's real world impact
  - View a full history of past donations, most recent first
  - Edit a donation's amount, which automatically regenerates the impact explanation to match the new amount
  - Delete a donation
- Full CRUD on donations: create, read, update, and delete, all scoped to the logged in user so nobody can view or modify another user's data
- Graceful error handling for the AI integration: if Gemini is temporarily unavailable, the donation still saves successfully and shows a "pending" status instead of blocking the user or crashing the app
- Responsive layout that adapts to mobile screen widths

## Tech stack

- **Backend:** Node.js, Express
- **Templating:** EJS
- **Database:** MongoDB Atlas, accessed through Mongoose
- **Authentication:** express-session for session management, bcryptjs for password hashing
- **AI API:** Google Gemini API (model: `gemini-2.5-flash`, with automatic fallback to `gemini-2.0-flash` if the primary model is unavailable)
- **Hosting:** Render (free tier)

## Database schema

Two MongoDB collections, connected through a reference field.

### `users`

| Field | Type | Notes |
|---|---|---|
| `username` | String | required |
| `email` | String | required, unique, used to log in |
| `passwordHash` | String | required, bcrypt hash, never the raw password |
| `createdAt` | Date | defaults to time of creation |

### `donations`

| Field | Type | Notes |
|---|---|---|
| `userId` | ObjectId | required, references `users._id` |
| `cause` | String | required, e.g. "Clean Water" |
| `amount` | Number | required, USD amount donated |
| `impactText` | String | nullable, the AI generated explanation; null until Gemini responds successfully |
| `createdAt` | Date | defaults to time of creation |

## How the AI integration works

When a donation is logged or edited, the server sends the cause and amount to Gemini with a prompt asking for a short, plain language explanation of what that donation could realistically accomplish. The response is parsed defensively (checked for missing or malformed data at every step) and saved alongside the donation as `impactText`.

If Gemini fails for any reason (rate limit, temporary outage, network issue), the function retries briefly, then falls back to a secondary model, and ultimately returns `null` rather than throwing an error. The donation always saves successfully either way; the dashboard simply shows "Impact summary coming soon" until a successful response is available.

## Running locally

1. Clone the repo and install dependencies:
   ```
   git clone https://github.com/kazi011225/CashLift-Final-Project.git
   cd CashLift-Final-Project
   npm install
   ```

2. Copy `.env.example` to `.env`:
   ```
   cp .env.example .env
   ```

3. Fill in your own values in `.env`:
   - `MONGODB_URI`: a MongoDB Atlas connection string (free tier cluster works fine)
   - `SESSION_SECRET`: any long random string
   - `GEMINI_API_KEY`: an API key from Google AI Studio

4. Start the server:
   ```
   npm start
   ```

5. Visit `http://localhost:3000`
